# Static Layer

Статический слой проекта собирается здесь до этапа WordPress-интеграции.

Главный принцип: секция живет автономно.

- страничный `pug` в `template-parts/pug/pages/` компилируется в корень `static/` как обычный `.html`
- у каждой секции свой `pug`-файл в `template-parts/pug/section/<section>/<section>.pug`
- у каждой секции свой partial SCSS-файл в `scss/section/<section>/_<section>.scss`
- Prepros компилирует секционный `pug` в `template-parts/html/section/<section>/<section>.html`
- на этапе статической разработки стили собираются через общий `scss/style.scss`
- секционные partial-стили подключаются в `style.scss` через `@import`, чтобы код оставался разложенным по блокам
- `._container` не стилизуем: контейнер отвечает только за общую ширину и отступы, а секционная раскладка живет во внутреннем `<section-name>__wrapper`

Базовая схема:

- `template-parts/pug/` — исходные Pug-шаблоны страниц, секций и включаемых частей
- `template-parts/html/` — HTML-выгрузки секций и компонентов
- `scss/` — исходники стилей
- `css/` — скомпилированные CSS-файлы
- `js/` — фронтовые скрипты и модули
- `images/`, `webp-images/`, `img/`, `fonts/` — ассеты

Правило слоя:

1. Сначала собираем и стабилизируем статику здесь.
2. Страница собирается из секций.
3. Только потом переносим готовые секции в WordPress-тему.

UI-слой сайта зафиксирован отдельно в `UI-SYSTEM.md`.
