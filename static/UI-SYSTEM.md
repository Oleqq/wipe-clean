# UI System

Ниже зафиксирован общий UI-слой сайта, собранный по всем переданным страницам Figma. Его задача: не плодить похожие блоки с разным неймингом и не дублировать один и тот же CSS по секциям.

## Базовое правило

- `ui-*` — только для глобально переиспользуемых элементов и паттернов
- `<section-name>*` — только для секционной логики и уникальной раскладки
- `._container` — только для общих контейнерных ограничений, раскладку секции делаем на внутреннем `<section-name>__wrapper`
- если меняется только цвет, размер, плотность или выравнивание, это модификатор, а не новый блок
- если меняется семантика и структура, это новый блок, но он должен собираться из `ui-*`

## Зафиксированные глобальные блоки

- `ui-section`, `ui-section--compact`, `ui-section--wave`
- `ui-section-head`, `ui-section-head--center`, `ui-section-head--compact`, `ui-section-head--narrow`
- `ui-kicker`
- `ui-title`, `ui-title--hero`, `ui-title--card`, `ui-title--compact`, `ui-title--number`
- `ui-text`, `ui-text--sm`, `ui-text--soft`
- `ui-btn`, `ui-btn--primary`, `ui-btn--secondary`, `ui-btn--accent`, `ui-btn--small`, `ui-btn--full`, `ui-btn--icon`
- `ui-card`, `ui-card--compact`, `ui-card--feature`, `ui-card--center`, `ui-card--faq`
- `ui-price-list`
- `ui-form`, `ui-form__row`
- `ui-field`
- `ui-choice`, `ui-choice--active`
- `ui-checkbox`
- `ui-media`
- `ui-icon-badge`
- `ui-socials`
- `ui-banner`

## Что использовать дальше по макетам

- hero, intro, SEO-текст, CTA-зоны:
  использовать `ui-section-head`, `ui-kicker`, `ui-title`, `ui-text`, `ui-btn`
- карточки услуг, акций, новостей, преимуществ:
  начинать с `ui-card`, а различия оформлять модификаторами и секционными элементами
- FAQ:
  использовать `ui-card ui-card--faq`
- формы расчета и обратной связи:
  использовать `ui-form`, `ui-field`, `ui-choice`, `ui-checkbox`, `ui-btn`
- контакты и соцсети:
  использовать `ui-card ui-card--compact`, `ui-socials`, `ui-btn--icon`
- ценовые списки:
  использовать `ui-price-list`

## Когда создавать новый глобальный блок

Создаем новый `ui-*` блок только если паттерн повторяется минимум на двух страницах и имеет устойчивую структуру. Если паттерн живет на одной секции, он остается секционным блоком.

## Что не делать

- не вводить новые кнопки без проверки, покрывает ли кейс `ui-btn` + модификатор
- не называть одинаковые карточки по-разному в разных секциях
- не копировать стили полей формы внутрь секций
- не смешивать глобальные `ui-*` классы и уникальные классы секции в одну сущность без причины
