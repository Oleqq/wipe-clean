# Набор агентов для проекта `a4-remont`

Этот каталог содержит агентские markdown-доки для LLM-разработки.

Главный принцип проекта: работа идет в **два разных этапа**, и агент должен это понимать сразу.

## Два этапа проекта

### 1. Верстка

Сначала весь сайт собирается как **полная статика** в корне проекта, в каталоге `static/`.

Это отдельный слой разработки:

- страницы собираются как статические HTML/Pug-страницы;
- секции собираются как самостоятельные статические блоки;
- стили пишутся в `static/scss`;
- JS-поведение сначала решается на статическом слое;
- именно `static/` является первым источником структуры нового макета.

На этом этапе задача агента:

- сверстать новый макет;
- подготовить секции и страницы в системе проекта;
- не думать раньше времени про ACF, CPT, flexible content и PHP-шаблоны.

### 2. Интеграция

Только после завершения статической сборки начинается **интеграция в WordPress-тему**.

Это уже второй, отдельный этап:

- перенос секций в `wp-content/themes/a4-remont/template-parts/section/`;
- добавление fallback-разметки в `template-parts/static/`;
- добавление ACF-полей через `inc/acf-*.php`;
- подключение CSS/JS темы;
- подключение секций в архивы, single, page-template, option pages и builders.

На этом этапе задача агента:

- не верстать макет заново;
- не придумывать новую структуру;
- аккуратно перенести уже готовую статику в архитектуру темы.

## Назначение набора

- ускорять понимание структуры проекта;
- помогать сначала собирать дизайн-макеты в полном static-слое;
- помогать потом интегрировать эту статику в WordPress-тему;
- не допускать хаотичных решений в ACF, CSS, JS и шаблонах.

Это не пользовательская инструкция и не редакторская документация.  
Это рабочий набор правил для разработки новых макетов, секций и страниц в методологии `a4-remont`.

## Базовый порядок работы по новому макету

1. Сначала открыть карту проекта: [00-project-structure-agent.md](./00-project-structure-agent.md).
2. Определить текущий этап:
   - если макет еще только собирается визуально, это этап **Верстка**;
   - если статика уже готова и переносится в тему, это этап **Интеграция**.
3. Использовать только соответствующую группу агентов, не смешивая этапы.

## Общий принцип

Любой агент должен сохранять совместимость с текущей архитектурой проекта:

1. Сначала формируется статический источник в `/static`.
2. Только потом этот источник интегрируется в тему.
3. Нельзя перескакивать сразу к WordPress-интеграции, если статический слой еще не собран.
4. Нельзя превращать WordPress-тему в место "черновой верстки".
5. Нельзя ломать контентные данные при обычной заливке обновленной темы на прод.
