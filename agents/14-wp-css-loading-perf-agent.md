# Агент: WP CSS Loading & Performance

## Роль

Поддерживает посекционную загрузку CSS в теме `a4-remont` без визуальных регрессий.

## Текущая модель загрузки

1. `fonts` + vendor (`swiper`)
2. `css/base.css`
3. `css/section/*.css` в порядке секций
4. `style.css` как override-слой

Контроллер: `wp-content/themes/a4-remont/inc/enqueue.php`.

## Алгоритм при добавлении/изменении секций

1. Добавить/обновить CSS-файл секции в `css/section/`.
2. Убедиться, что slug секции совпадает с шаблоном/layout map.
3. Если секция использует классы из другого чанка, добавить зависимость в:
   - `a4_remont_expand_section_style_dependencies()`.
4. Проверить порядок каскада и финальные подключенные стили.
5. Проверить консольный debug-блок `A4 CSS Debug`.

## Типовые проблемы и решения

- Проблема: секция на странице есть, а CSS нет.  
  Решение: проверить map layout -> template slug -> section slug -> имя файла.

- Проблема: стили «поехали» после отключения большого legacy CSS.  
  Решение: найти кросс-секционные зависимости и вынести их в dependency map или в `base.css`.

- Проблема: конфликт каскада между секцией и override.  
  Решение: править в секционном файле, а `style.css` оставлять только для целевых override.

## Definition of Done

- загружаются только нужные section CSS;
- критичные зависимости учтены;
- визуал совпадает со статикой и интеграцией.

## Жесткие правила

- не возвращать проект к монолитному `css/style.css` как основному источнику;
- не оставлять скрытые зависимости между секциями без явной карты;
- не удалять debug-инструмент до стабилизации страницы.
