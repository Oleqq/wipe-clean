# Агент: WP Section Integrator

## Роль

Это агент **этапа Интеграция**.

Он не верстает макет с нуля, а интегрирует уже собранную статику в тему `a4-remont` через `template-parts/section` с сохранением fallback-режима.

Перед началом работы агент должен исходить из того, что:

- статический источник уже существует в `/static`;
- структура секции уже определена на этапе верстки;
- задача здесь — перенос в WordPress, а не повторная фронтенд-разработка.

## Основные пути проекта

- секции PHP: `wp-content/themes/a4-remont/template-parts/section/`
- статические fallback: `wp-content/themes/a4-remont/template-parts/static/section/`
- helper-функции: `wp-content/themes/a4-remont/inc/section-helpers.php`
- fallback API: `wp-content/themes/a4-remont/inc/static-content.php`
- источник верстки: `static/`

## Алгоритм интеграции секции

1. Взять готовую секцию из `/static` как источник структуры.
2. Создать/обновить `template-parts/section/<section>.php`.
3. Реализовать ветку dynamic (ACF) и ветку fallback (static markup).
4. Для медиа использовать helper-функции темы (`a4_remont_get_acf_image_html` и т.п.).
5. Для кнопок использовать нормализаторы ссылок/действий (включая popup action).
6. Если секция отсутствует в данных ACF, возвращать fallback без фаталов.
7. Если секция уже есть в проекте, расширять ее, а не дублировать.

## Definition of Done

- секция корректно рендерится при заполненных и пустых полях;
- fallback не ломается;
- нет дублирования бизнес-логики в нескольких секциях.
- структура секции узнаваемо соответствует static-источнику.

## Жесткие правила

- не рендерить необработанный пользовательский HTML;
- не допускать обращения к несуществующим индексам массивов;
- не создавать копии одинаковых секций под разные страницы без причины.
- не превращать PHP-шаблон в место первичной верстки, если эта работа должна была быть сделана в `/static`.
