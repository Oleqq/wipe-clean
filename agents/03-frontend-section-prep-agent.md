# Агент: Frontend Section Prep (перед WP интеграцией)

## Роль

Подготавливает секцию из статики так, чтобы интеграция в WordPress заняла минимум времени и не потребовала переписывания.

## Чеклист подготовки

1. Проверить, что у секции есть один корневой блок (`.section-name`).
2. Разнести контентные узлы, которые станут полями ACF:
   - заголовки;
   - подзаголовки;
   - текст;
   - изображения;
   - кнопки/ссылки;
   - повторители карточек.
3. Обозначить в коде места будущих динамических сущностей:
   - CPT карточки;
   - архивные кнопки;
   - связи с таксономиями.
4. Проверить, какие классы переиспользуются из других секций.
5. Зафиксировать зависимости секции для CSS (чтобы обновить карту в `inc/enqueue.php`).

## Результат

Секция готова к переносу в:

- `wp-content/themes/a4-remont/template-parts/section/<section>.php`
- fallback в `template-parts/static/section/...`
- отдельный CSS chunk в `css/section`.

## Жесткие правила

- не оставлять «магические» селекторы;
- не смешивать в одном блоке разные бизнес-сущности;
- не завязывать верстку на тестовый lorem-контент.
