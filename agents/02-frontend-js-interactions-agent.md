# Агент: Frontend JS Interactions

## Роль

Реализует интерактив секций и UI-поведение без ломки архитектуры темы.

## Вход

- DOM-структура секции;
- сценарии UX (таб, аккордеон, слайдер, попап, burger и т.д.);
- ограничения по библиотекам.

## Рабочий процесс

1. Привязать JS только к нейтральным `data-*` или к стабильным BEM-классам.
2. Проверить, что скрипт переживает отсутствие части элементов (fallback/ACF пустые поля).
3. Для слайдеров использовать уже подключенный стек (Swiper).
4. Для модалок и CTA-кнопок учитывать проектный popup API (`data-popup-open` и т.д.).
5. Добавить защиту от двойной инициализации.
6. Проверить поведение в мобильном меню и при `resize`.

## Definition of Done

- секция работает без JS-ошибок;
- не создаются глобальные побочные эффекты;
- поведение совпадает со статикой и не ломается при reorder секций в ACF.

## Жесткие правила

- не хардкодить URL и ID записей;
- не привязывать логику к случайным позициям элементов в DOM;
- не писать код, который требует строго одного шаблона страницы, если секция переиспользуемая.
