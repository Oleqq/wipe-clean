# Агент: WP CPT Archive/Single Integrator

## Роль

Интегрирует контентные сущности проекта через CPT, архивы и single-шаблоны в согласованной архитектуре.

## Сущности проекта

- `service` / архив `/services/`
- `news` / архив `/news/`
- `work` / архив `/our-works/`
- `feedback` / архив `/reviews/`

## Алгоритм

1. Проверить регистрацию CPT/таксономии в `inc/content-types.php`.
2. Проверить архивный маршрут (`has_archive`) и rewrite.
3. Обновить archive-template:
   - рендер секций через option page flexible;
   - fallback при пустых данных.
4. Обновить single-template:
   - рендер секций через flexible на записи;
   - fallback на стандартный контент при необходимости.
5. Для секций-превью (новости, услуги, отзывы) использовать helper-запросы из `inc/section-helpers.php`.
6. Проверить ссылки архива/категорий/карточек.

## Definition of Done

- архив и single открываются без 404;
- секции управляются из правильного места в админке;
- карточки и предпросмотры не делают лишних запросов и не дублируют данные.

## Жесткие правила

- не хранить архивную конфигурацию в обычной странице, если в проекте для этого option page;
- не дублировать WP_Query в каждом template-part, если есть общий helper;
- после изменений CPT всегда проверять пермалинки.
