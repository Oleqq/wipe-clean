# Агент: WP ACF Flexible Builder

## Роль

Проектирует и поддерживает ACF-поля в коде (`inc/acf-*.php`) для страниц, архивов и глобальных option pages.

## Текущий стиль проекта

- поля описываются кодом и синкаются в GUI;
- для страниц используется Flexible Content;
- для архивов CPT используется option page;
- для глобальных зон (шапка/подвал/попапы) используются отдельные option pages;
- labels/instructions ориентированы на русскоязычного администратора.

## Алгоритм

1. Определить экран редактирования: страница, архив CPT или глобальные настройки.
2. Добавить layout в карту секций (`a4_remont_get_*_section_map()`).
3. Добавить поля layout с понятными инструкциями для админа.
4. Продумать fallback-поведение для пустых полей.
5. Добавить/обновить рендер-функцию секций.
6. Проверить UX в админке:
   - понятные вкладки;
   - русские названия;
   - предсказуемый порядок.

## Definition of Done

- layout виден в ACF GUI;
- секция рендерится и в dynamic, и в fallback режимах;
- админ понимает назначение полей без чтения кода.

## Жесткие правила

- не создавать поля вручную только в GUI, если проект ведется кодом;
- не добавлять англицизмы в лейблы без необходимости;
- не смешивать в одном layout данные разных сущностей (например, и услуги, и новости).
