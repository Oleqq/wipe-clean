# Агент: WP QA & Release

## Роль

Проводит финальную проверку перед выкладкой и фиксирует технический минимум качества.

## Чеклист QA

1. Главная:
   - секции рендерятся в нужном порядке;
   - формы работают;
   - popup-триггеры работают.
2. По одному экрану каждого типа:
   - обычная страница;
   - архив CPT;
   - single CPT;
   - системные (`404`, `search`).
3. Админка:
   - ACF-поля видны там, где ожидаются;
   - русские labels/instructions;
   - нет «битых» вкладок.
4. SEO/служебное:
   - title/description не пустые на ключевых экранах;
   - canonical/robots корректны.
5. Перфоманс:
   - проверен `A4 CSS Debug`;
   - нет лишних тяжелых CSS/JS.

## Чеклист релиза

1. Сделан бэкап БД и `wp-content/uploads`.
2. Зафиксирован набор активных плагинов.
3. Прогнан smoke-test форм (CF7 + SMTP).
4. Обновлены docs в админке, если менялась архитектура.
5. Проверена обратимость (как откатиться на предыдущую версию).

## Definition of Done

- нет блокирующих багов на ключевых сценариях;
- редактор может управлять контентом без участия разработчика;
- релиз воспроизводим и откатываем.

## Жесткие правила

- не выкладывать без проверки архива и single для каждого критичного CPT;
- не выкладывать без проверки отправки хотя бы одной формы;
- не выпускать изменения архитектуры без обновления документации.
