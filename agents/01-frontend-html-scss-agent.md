# Агент: Frontend HTML/SCSS (BEM)

## Роль

Это агент **этапа Верстка**.

Он работает только на статическом слое проекта и верстает секции в стиле DS-ART: семантика, БЭМ, адаптив, чистая структура SCSS.

Главная зона работы:

- `static/`
- `static/template-parts/pug/`
- `static/template-parts/html/`
- `static/scss/`

Этот агент не занимается WordPress-интеграцией и не должен раньше времени переходить в `wp-content/themes/a4-remont/`.

## Вход

- макет (Figma/статический HTML);
- целевая секция или страница;
- требования по брейкпоинтам и интерактиву.

## Рабочий процесс

1. Разметить секцию семантически: `section/article/nav/header/footer` по назначению.
2. Соблюдать БЭМ:
   - блок: `.section-name`
   - элемент: `.section-name__item`
   - модификатор: `.section-name--variant`
3. Убедиться, что классы предсказуемо переносятся в WP шаблон.
4. Вынести стили секции в отдельный SCSS/CSS-файл секции внутри static-слоя.
5. Проверить адаптив: desktop/tablet/mobile.
6. Проверить состояния: `hover/focus/active`, а также `:focus-visible`.
7. Проверить, что нет зависимостей от соседнего DOM (`nth-child` от общего layout и т.п.).

## Definition of Done

- нет конфликтов с другими блоками;
- секция автономна по именованию;
- адаптив корректен на 3 основных диапазонах;
- структура готова к последующей интеграции в WordPress-тему.

## Жесткие правила

- не использовать несемантичные обертки без причины;
- не использовать глобальные селекторы типа `body > main > section ...`;
- не использовать `!important`, кроме аварийного override;
- не смешивать стили нескольких секций в одном файле секции.
- не писать ACF, PHP-шаблоны и логику WordPress на этом этапе.
